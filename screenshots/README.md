# Dossier des Captures d'Écran

Ce dossier contient les captures d'écran pour illustrer le rapport de sécurité.

## Format recommandé
- **Format** : PNG ou JPG
- **Nommage** : Utilisez des noms descriptifs, par exemple :
  - `sql-injection-attempt.png`
  - `password-hash-comparison.png`
  - `authentication-error.png`
  - `unauthorized-access.png`

## Comment ajouter une image dans le rapport

Dans le fichier `RAPPORT_SECURITE.md`, utilisez la syntaxe Markdown suivante :

```markdown
![Description de l'image](screenshots/nom-du-fichier.png)
```

### Exemples d'emplacements suggérés dans le rapport :

1. **Section Injection SQL** : Capture d'une tentative d'injection SQL
2. **Section Authentification** : Capture d'une erreur d'authentification
3. **Section Validation** : Capture d'un message d'erreur de validation
4. **Section Tests** : Captures des résultats des tests

## Redimensionnement

Si vos images sont trop grandes, vous pouvez les redimensionner avec un outil comme :
- Paint (Windows)
- GIMP
- ImageMagick (ligne de commande)

Une largeur de 800-1200 pixels est généralement appropriée pour un rapport.
